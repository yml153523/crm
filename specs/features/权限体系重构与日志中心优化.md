# 权限体系重构与系统日志中心优化

**版本**: v1.0  
**日期**: 2026-04-15  
**状态**: 待确认

---

## 1. 功能概述

### 1.1 背景

当前系统存在以下问题：
1. **功能冗余**: "审计日志"和"日志统计"是同一功能的不同视图，造成用户困惑
2. **权限缺失**: 缺少明确的超级管理员/管理员权限划分
3. **管理功能不足**: 无法通过界面创建和管理管理员账号
4. **UI体验待提升**: 页面布局和交互需要优化

### 1.2 目标

构建**清晰的权限体系**和**统一的日志管理中心**:
- 合并审计日志和统计为单一入口
- 实现三级权限控制（super_admin / admin / user）
- 新增管理员CRUD管理功能
- 优化UI/UX，提升专业性

### 1.3 核心价值

| 用户类型 | 核心价值 |
|----------|----------|
| **超级管理员** | 完全掌控系统，可审计一切操作，管理团队 |
| **管理员** | 高效工作，查看所需日志，专注业务 |
| **系统** | 安全可控，操作留痕，权限分明 |

## 3. 功能需求详解

### 3.1 模块一：系统日志中心（合并优化）

#### 3.1.1 功能定位

将原来的"审计日志"和"日志统计"合并为**统一的系统日志中心**，采用Tab切换方式展示不同视角。

#### 3.1.2 页面结构

```
┌─────────────────────────────────────────────────────┐
│  📊 系统日志中心              [导出CSV] [刷新]      │
├─────────────────────────────────────────────────────┤
│  [📋 日志列表] [📈 数据统计] [⚙️ 日志管理*]        │
├─────────────────────────────────────────────────────┤
│                                                     │
│  ┌─ Tab内容区域 ─────────────────────────────────┐ │
│  │                                               │ │
│  │  (根据选中的Tab显示不同内容)                    │ │
│  │                                               │ │
│  └───────────────────────────────────────────────┘ │
│                                                     │
└─────────────────────────────────────────────────────┘

* "日志管理"Tab仅对 super_admin 可见
```

#### 3.1.3 Tab1: 日志列表（所有角色可用）

**功能特性:**
- ✅ 多维度筛选（时间范围、操作类型、用户、成功/失败状态）
- ✅ 关键词全文搜索
- ✅ 分页浏览（每页20条，支持切换）
- ✅ 日志详情弹窗（点击单条查看完整信息）
- ✅ 导出当前筛选结果为CSV（admin及以上）

**显示字段:**
| 字段 | 说明 | 示例 |
|------|------|------|
| 时间 | 操作时间 | 2026-04-15 08:30:15 |
| 类型 | 操作类型图标+文字 | 🔵 登录 |
| 操作者 | 用户名+手机号 | 张三 / 138****8000 |
| 操作描述 | 资源信息 | 登录系统 |
| IP地址 | 来源IP | 192.168.1.100 |
| 设备信息 | 设备类型+浏览器 | 💻 PC / Chrome |
| 状态 | 成功/失败标识 | ✅ 或 ❌ |
| 失败原因 | 失败时显示 | 密码错误（可选） |

**验收标准:**
- AC-LOG-001: Given 管理员已登录 When 访问日志列表 Then 显示最近20条日志，按时间倒序
- AC-LOG-002: Given 在日志列表页 When 选择时间范围"今天"并点击查询 Then 只显示今天的日志
- AC-LOG-003: Given 在日志列表页 When 输入关键词"张三"并搜索 Then 显示包含"张三"的日志
- AC-LOG-004: Given 在日志列表页 When 点击第2页 Then 显示第21-40条日志
- AC-LOG-005: Given 在日志列表页 When 点击单条日志 Then 弹窗显示完整详情
- AC-LOG-006: Given admin角色 When 点击导出CSV Then 下载当前筛选结果的CSV文件
- AC-LOG-007: Given 无日志数据时 Then 显示友好的空状态提示

#### 3.1.4 Tab2: 数据统计（所有角色可用）

**功能特性:**
- ✅ 今日概览卡片（登录次数、操作次数、错误数、趋势）
- ✅ 近7天趋势图（柱状图：登录vs操作双线对比）
- ✅ 操作类型分布（横向进度条+百分比）
- ✅ 活跃用户TOP10排行榜
- ✅ 快捷跳转（点击统计数据可跳转到对应筛选条件的列表）

**统计指标:**
| 指标 | 说明 | 更新频率 |
|------|------|----------|
| 今日登录数 | 当日登录成功次数 | 实时 |
| 今日操作数 | 当日所有操作次数 | 实时 |
| 今日错误数 | 当日失败操作数 | 实时 |
| 较昨日增长 | 与昨日对比的百分比 | 实时 |
| 7日趋势 | 最近7天的数据走势 | 实时 |
| 操作分布 | 各操作类型占比 | 实时 |
| 活跃用户 | 操作次数最多的10个用户 | 实时 |

**验收标准:**
- AC-STAT-001: Given 管理员已登录 When 切换到"数据统计"Tab Then 显示今日概览卡片
- AC-STAT-002: Given 在统计页 When 查看今日登录数 Then 显示准确的数字（如128次）
- AC-STAT-003: Given 在统计页 When 查看7日趋势图 Then 显示柱状图，横轴为日期，纵轴为次数
- AC-STAT-004: Given 在统计页 When 查看操作类型分布 Then 显示各类型的占比进度条
- AC-STAT-005: Given 在统计页 When 点击"登录次数"卡片 Then 跳转到日志列表并自动筛选LOGIN类型

#### 3.1.5 Tab3: 日志管理（仅super_admin可见）

**功能特性:**
- ✅ 日志清理（按时间范围批量删除旧日志）
- ✅ 日志归档（导出并标记为已归档）
- ✅ 保留策略配置（最少保留天数，默认7天）
- ✅ 清理预览（删除前显示将删除的日志数量）
- ✅ 二次确认机制（防止误操作）

**安全约束:**

- 删除操作本身会被记录到日志
- 需要二次确认（输入"CONFIRM"或勾选确认框）

**验收标准:**
- AC-MGMT-001: Given super_admin已登录 When 切换到"日志管理"Tab Then 显示日志管理面板
- AC-MGMT-002: Given admin角色 When 访问系统日志 Then 不显示"日志管理"Tab

#### 3.2.3 功能清单

**1. 管理员列表**
- 显示所有管理员账号（除自己外）
- 支持按状态筛选（全部/启用/禁用）
- 显示关键信息：头像、用户名、手机号（脱敏）、角色、状态、最后登录时间
- 卡片式布局，每张卡片包含操作按钮

**2. 创建管理员**
- 点击"+ 创建管理员"按钮打开表单
- 表单字段:
  - 用户名*（必填，2-20字符，唯一）
  - 手机号*（必填，11位手机号格式，唯一）
  - 密码*（必填，8-32位，需包含字母和数字）
  - 确认密码*（必填，需与密码一致）
  - 角色*（选择：admin）
  - 权限（多选框：会员管理、课程管理、视频管理等）
  - 备注（选填，200字以内）
- 表单验证：实时验证，提交前再次校验
- 创建成功后：
  - 显示成功提示
  - 自动刷新列表
  - 可选择是否发送通知给新管理员

**3. 编辑管理员**
- 点击卡片的"编辑"按钮
- 可编辑字段：权限、备注、角色
- 不可编辑字段：用户名、手机号（如需修改需联系超级管理员手动操作）
- 保存后刷新列表

**4. 禁用/启用管理员**
- 点击"禁用"按钮 → 二次确认 → 禁用该账号
- 禁用后：
  - 该管理员无法登录后台
  - 显示"已禁用"状态标签
  - 可点击"启用"恢复
- 不能禁用自己（当前登录的super_admin）

**5. 重置密码**
- 点击"重置密码"按钮 → 输入新密码 → 确认 → 重置
- 重置后提示"密码已重置，请通知该管理员"
- 建议发送短信/邮件通知（可选，后续实现）

**6. 删除管理员**
- 危险操作，需特别谨慎
- 点击"删除" → 输入管理员用户名确认 → 二次确认 → 软删除
- 软删除：标记为deleted，不物理删除（保留日志关联）
- 删除后从列表移除

**验收标准:**
- AC-ADMIN-001: Given super_admin已登录 When 访问管理员管理 Then 显示所有管理员列表
- AC-ADMIN-002: Given admin角色 When 访问管理员管理 Then 显示"无权限"或隐藏入口
- AC-ADMIN-003: Given 在管理员管理页 When 点击"创建管理员" Then 显示创建表单
- AC-ADMIN-004: Given 在创建表单 When 输入有效信息并提交 Then 创建成功并刷新列表
- AC-ADMIN-005: Given 在创建表单 When 输入已存在的手机号 Then 提示"该手机号已被注册"
- AC-ADMIN-006: Given 在管理员列表 When 点击"禁用"并确认 Then 该管理员状态变为"已禁用"
- AC-ADMIN-007: Given 在管理员列表 When 对自己点击"禁用" Then 提示"不能禁用自己的账号"
- AC-ADMIN-008: Given 在管理员列表 When 点击"重置密码"并输入新密码 Then 密码重置成功
- AC-ADMIN-009: Given 在管理员列表 When 点击"删除"并确认 Then 管理员被软删除

## 4. 技术约束

### 4.1 后端API要求

**已有的API（无需修改）:**
- `GET /api/audit-logs` - 查询日志列表
- `GET /api/audit-logs/:id` - 日志详情
- `GET /api/audit-logs/statistics/overview` - 统计数据
- `POST /api/audit-logs/export` - 导出CSV
- `DELETE /api/audit-logs/cleanup` - 清理日志（已有权限控制）

**需要新增的API:**
- `GET /api/admin/users` - 获取管理员列表
- `POST /api/admin/users` - 创建管理员
- `PUT /api/admin/users/:id` - 编辑管理员
- `PUT /api/admin/users/:id/status` - 启用/禁用
- `PUT /api/admin/users/:id/password` - 重置密码
- `DELETE /api/admin/users/:id` - 删除管理员（软删除）
- `GET /api/admin/users/:id` - 管理员详情

### 4.2 权限控制实现

**前端权限控制:**
- 根据登录用户的 `role` 字段判断
- 使用路由守卫或条件渲染控制页面/组件显示
- API请求携带Token，后端验证权限

**后端权限控制:**
- 已实现JWT中间件 (`authenticateToken`)
- 已实现RBAC中间件 (`validateAdminRole`, `validateSuperAdminRole`)
- 新增的管理员管理API需添加 `validateSuperAdminRole`

### 4.3 数据库变更

**User模型可能需要的更新:**
```javascript
// 可能需要新增的字段
{
  role: { type: String, enum: ['super_admin', 'admin', 'user'], default: 'user' },

  permissions: [{ type: String }], // 自定义权限列表
  createdBy: { type: ObjectId, ref: 'User' }, // 创建者（谁创建的这个管理员）
  lastLoginAt: Date,
  note: String // 备注
}
```

## 5. UI/UX设计规范

### 5.1 设计原则

1. **一致性**: 遵循项目现有的UI设计规则（[ui-design-rules.md](../../.trae/rules/ui-design-rules.md)）
2. **简洁性**: 减少重复入口，合并相似功能
3. **层次感**: 通过颜色、字号、间距区分信息层级
4. **反馈性**: 所有操作都有即时反馈（loading、success、error）
5. **容错性**: 友好的空状态、错误提示、边界处理

### 5.2 色彩规范（沿用项目规范）

| 用途 | 颜色 | 说明 |

| 成功 | `#34C759` | 成功状态、启用 |

### 5.3 组件规范

- **按钮**: 高度44-48px，圆角8-12px
- **输入框**: 高度44-48px，圆角8-12px
