**优先级**: P0 (最高)

## 1. 背景与目标

### 1.1 业务背景

当前CRM系统存在两个端口：
- **8081端口** (管理后台): 管理员使用PC端管理视频、商品、课程、推荐策略、红包活动
- **8080端口** (用户端): 普通用户使用手机端浏览内容、下单购买、领取红包

### 1.2 核心问题

当前8080端口的用户体验不够优化：
1. 登录流程繁琐
2. 推荐内容展示不突出
3. 从浏览到下单的转化路径过长
4. 红包领取流程不够直观

### 1.3 项目目标

**核心原则**: "怎么简单怎么来" - 极致简化用户体验

**量化目标**:
| 指标 | 当前 | 目标 | 提升 |
|------|------|------|------|
| 登录完成时间 | >60秒 | ≤30秒 | -50% |
| 推荐内容点击率 | 基准 | +20% | +20% |
| 浏览→下单步骤数 | 7步 | ≤5步 | -30% |
| 红包领取转化率 | 基准 | 预期目标 | 待定 |

## 2. 用户角色与场景

### 2.1 用户角色

| 角色 | 使用端口 | 设备 | 核心任务 |
|------|---------|------|---------|
| **管理员** | 8081 | PC/平板 | 管理 内容 / 设置推荐 / 创建红包 |
| **普通用户** | 8080 | 手机 | 浏览 / 观看视频 / 下单 / 领红包 |

### 2.2 核心用户场景

#### 场景A: 新用户首次访问
```
用户打开手机 → 访问8080 → 看到精彩内容 → 
被吸引 → 手机号登录 → 获得新用户红包 → 
浏览推荐 → 下单购买
```

#### 场景B: 老用户回访
```
用户打开8080 → 自动登录(记住状态) → 
看到个性化推荐(基于历史) → 领取今日红包 → 
继续浏览 → 下单
```

#### 场景C: 从分享链接进入
```
用户在微信/短信收到链接 → 点击 → 
跳转8080(带上下文) → 看到具体商品/视频 → 
感兴趣 → 登录 → 领红包优惠 → 下单
```

## 4. 详细功能需求与验收标准

### 4.1 登录模块

#### 4.1.1 手机号快速登录

**功能描述**:  
用户通过手机号+验证码完成登录，整个流程不超过4步，耗时≤30秒。

**UI设计要求**:
- 全屏登录页，背景使用品牌渐变色(#667eea → #764ba2)
- 顶部显示Logo + Slogan("让学习更简单")
- 手机号输入框(大字体,自动聚焦)
- 验证码输入框(6位数字,单独格子显示)
- "获取验证码"按钮(60秒倒计时)
- "登录"按钮(全宽,渐变背景,圆角24rpx)
- 底部显示"已阅读并同意《用户协议》"

**验收标准 (AC)**:

**AC-001: 正常登录流程**
```
Given 用户在未登录状态访问8080端口
When 输入正确的手机号(11位)
And 点击"获取验证码"
Then 系统发送验证码到该手机号(5秒内到达)
And 按钮变为倒计时"59s后重发"(禁用状态)

When 输入收到的6位验证码
And 点击"登录"按钮
Then 页面跳转到首页(加载时间<1s)
And 顶部显示"欢迎回来, [昵称]"
And 弹出"新用户专享红包"弹窗
```

**AC-002: 验证码错误处理**
```
Given 用户在登录页面
When 输入错误的验证码(6位数字但不匹配)
And 点击"登录"
Then 验证码输入框抖动动画(红色边框)
And 显示错误提示"验证码错误,请重新输入"(2秒后消失)
And 不清空输入框(方便用户修改)
```

**AC-003: 手机号格式校验**
```
Given 用户在登录页面
When 输入非11位手机号(如"12345")
Then "获取验证码"按钮为禁用状态(灰色,不可点击)
And 手机号下方显示提示"请输入正确的11位手机号"
```

**AC-004: 验证码过期处理**
```
Given 用户收到验证码(有效期5分钟)
When 超过5分钟后才输入验证码
And 点击"登录"
Then 显示提示"验证码已过期,请重新获取"
And "获取验证码"按钮恢复可点击状态
```

**AC-005: 重复发送限制**
```
Given 用户点击"获取验证码"成功
When 在60秒内再次点击
Then 按钮保持禁用状态
And 显示倒计时"XXs后重发"
And 60秒后自动恢复可点击
```

#### 4.1.2 自动登录(记住状态)

**功能描述**:  
登录成功后,记住用户状态7天,下次访问无需重新登录。

**验收标准 (AC)**:

**AC-006: 登录状态保持**
```
Given 用户成功登录
When 关闭浏览器/APP
And 在7天内重新访问8080端口
Then 自动进入首页(无需登录页)
And 显示用户头像和昵称
And 所有功能正常可用
```

**AC-007: 登录态过期**
```
Given 用户上次登录超过7天
When 访问8080端口
Then 显示登录页面(不是首页)
And 可选择"重新登录"或"游客模式浏览"
```

```
Given 管理员在8081设置了3张活跃Banner
When 用户访问8080首页
Then Banner区域显示3张轮播图(间隔3秒自动切换)
And 每张图的标题/副标题清晰可见
And 当前激活的指示器高亮(白色)

**AC-009: Banner点击跳转**

Given 用户正在查看第2张Banner(新课推荐)

**AC-010: 无Banner时的降级**

Given 管理员未设置任何Banner(或全部过期)

### 4.4 下单流程优化

#### 4.4.1 简化结算页

**功能描述**:  
极简结算页面,只保留必要信息,减少用户决策成本。

**UI设计要求**:
- 商品清单: 缩略图+名称+规格+数量+价格(可修改数量)
- 优惠券: 可用优惠券列表(默认选最优)
- 红包: 显示可用红包余额,可选择抵扣
- 价格明细: 商品总价-优惠-红包=实付金额
- 底部: "提交订单"按钮(全宽,红色,大字)

**验收标准 (AC)**:

**AC-022: 结算页加载**
```
Given 用户购物车有2件商品(共¥378)
When 进入结算页
Then 显示:
   - 商品清单(图片+名称+价格,可调数量)
   - 优惠券选择(默认最优省¥20)
   - 红包抵扣(可选使用¥5红包)
   - 价格明细: ¥378 - ¥20(券) - ¥5(红包) = **¥353**
   - "提交订单"按钮(醒目)
And 整个页面加载时间<1.5秒
```

**AC-023: 提交订单**
```
Given 用户确认订单信息无误
When 点击"提交订单"
Then 显示支付方式选择:
   - 微信支付(推荐)
   - 支付宝
   - 余额支付(如果有余额)
If 选择微信支付:
   Then 调起微信支付SDK(或显示二维码)
   And 支付成功后跳转到"订单详情"页
   And 显示"支付成功"状态+订单号
```

### 4.5 红包系统完整流程

#### 4.5.1 红包类型与规则

**功能描述**:  
实现多种类型的红包,满足不同营销场景。

| 红包类型 | 触发条件 | 金额范围 | 使用限制 |
|---------|---------|---------|---------|
| **新用户注册红包** | 首次登录 | ¥1-10 | 仅首单可用 |
| **每日签到红包** | 每日首次访问 | ¥0.1-2 | 当日有效 |
| **推荐裂变红包** | 分享给好友 | ¥1-5 | 好友注册后获得 |
| **满减红包** | 订单满X元 | ¥5-50 | 有门槛限制 |

**验收标准 (AC)**:

**AC-024: 新用户红包**
```
Given 新用户(从未登录)首次完成登录
When 登录成功跳转到首页

   - 显示红包金额(随机¥1-10)
