# CRM 系统技术栈选型文档

## 项目背景

本项目为**客户关系管理（CRM）系统**，需要实现**三合一全端覆盖**：
- ✅ 微信小程序
- ✅ Android/iOS APP
- ✅ H5 网页（手机 + PC）

**核心目标：一套代码，全端运行**

---

## 一、前端技术栈

### 1.1 核心框架：UniApp (Vue3)

| 技术项 | 选型 | 版本 | 说明 |
|--------|------|------|------|
| 框架 | **UniApp** | ^3.0+ | 跨平台开发框架，DCloud 出品 |
| 语言 | **Vue 3** | ^3.4+ | Composition API + `<script setup>` |
| 构建 | **Vite** | ^5.x | 极速构建工具 |

#### 为什么选择 UniApp + Vue3？

1. **真·跨平台**：一套代码可编译为微信小程序、支付宝小程序、H5、Android APP、iOS APP
2. **生态成熟**：国内 90%+ 移动端跨平台项目首选方案
3. **开发效率高**：比原生开发快 3-5 倍
4. **企业级验证**：大量商用 CRM 系统使用此方案

### 1.2 UI 组件库

**最终选择：uView Plus**
- 组件最全面（60+ 组件）
- 文档清晰，示例丰富
- 支持暗色模式

### 1.3 状态管理

**最终选择：Pinia**
- Vue 3 官方推荐，更简洁的 API
- 支持 Composition API

### 1.4 网络请求

**最终选择：uni.request 封装**
- 原生能力，无额外依赖
- 支持请求/响应拦截器

## 二、后端技术栈（保持不变）

| 技术项 | 选型 | 版本 |
|--------|------|------|
| 运行时 | **Node.js** | ^18+ LTS |
| 框架 | **Express** | ^4.x |
| 数据库 | **MongoDB** | ^6.x |
| ODM | **Mongoose** | ^8.x |

> 后端保持现有架构不变，仅重构前端为 UniApp + Vue3

## 三、代码映射关系

| React Native (旧) | UniApp + Vue3 (新) |
|-------------------|---------------------|
| `View` | `<view>` |
| `Text` | `<text>` |
| `StyleSheet.create()` | SCSS |
| `useState` | `ref()` / `reactive()` |
| `useEffect` | `onMounted` / `onUnmounted` |
| `Navigation` | `uni.navigateTo` |
| `Redux` | `Pinia Store` |
| `axios` | `uni.request` 封装 |

## 四、最终确认

✅ **前端**：UniApp + Vue3 + Vite + uView Plus + Pinia
✅ **后端**：Node.js + Express + MongoDB（保持不变）
✅ **目标平台**：微信小程序 + Android/iOS APP + H5 网页
