## 技术亮点

### 1. 异步批量写入

```javascript
// 队列机制
const logQueue = [];
const BATCH_SIZE = 10;
const FLUSH_INTERVAL = 5000;

// 批量写入
async function flushLogs() {
  const logsToWrite = [...logQueue];
  logQueue.length = 0;
  
  try {
    await AuditLog.insertMany(logsToWrite);
  } catch (error) {
    // 降级为单条写入
    for (const log of logsToWrite) {
      await new AuditLog(log).save();
    }
  }
}
```

### 2. 智能设备识别

```javascript
function getDeviceType(agent) {
  const ua = agent.source.toLowerCase();
  
  // 平板优先匹配
  if (/tablet|ipad|android(?!.*mobile)/.test(ua)) {
    return 'Tablet';
  }
  
  // 移动设备
  if (/mobile|android|iphone|ipod|blackberry|windows phone/.test(ua)) {
    return 'Mobile';
  }
  
  // 默认 PC
  return 'PC';
}
```

### 3. 多层脱敏保护

```javascript
function desensitize(data) {
  // 1. 删除敏感字段
  sensitiveFields.forEach(field => delete data[field]);
  
  // 2. 手机号脱敏
  data.phone = data.phone.replace(/(\d{3})\d{4}(\d{4})/, '$1****$2');
  
  // 3. 身份证脱敏
  data.idCard = data.idCard.replace(/(\d{6})\d{8}(\d{4})/, '$1********$2');
  
  // 4. 邮箱脱敏
  data.email = desensitizeEmail(data.email);
  
  return data;
}
```

### 4. 环境自适应

## 遇到的问题及解决方案

### 问题 1: 测试超时

**症状**: 测试运行超过 60 秒

**原因**: MongoDB Memory Server 占用大量内存，加上 Jest 的 coverage 收集

**解决**:
1. 创建简化版测试文件
2. 禁用 coverage 收集
3. 测试环境使用同步写入

### 问题 2: 中间件无法获取用户信息

**原因**: 中间件在认证中间件之前执行

**解决**: 确保中间件注册在认证中间件之后：
```javascript
app.use(authMiddleware);      // 先认证
app.use(auditLogMiddleware);  // 后日志
```

### 问题 3: 设备解析不准确

**原因**: User-Agent 字符串格式多样

**解决**:
1. 使用成熟的 useragent 库
2. 优先匹配平板设备（避免误判为手机）
3. 提供默认值处理未知设备
