## 技术亮点

### 1. 自动脱敏机制

```javascript
phone: {
  type: String,
  set: function(v) {
    // 手机号脱敏：13800138000 -> 138****8000
    if (v && typeof v === 'string' && v.length === 11) {
      return v.replace(/(\d{3})\d{4}(\d{4})/, '$1****$2');
    }
    return v;
  }
}
```

### 2. 敏感字段过滤

```javascript
actionParams: {
  type: Object,
  set: function(v) {
    // 删除敏感字段
    if (v && typeof v === 'object') {
      const sensitiveFields = ['password', 'token', 'secret', 'accessToken', 'refreshToken'];
      const cleaned = { ...v };
      sensitiveFields.forEach(field => {
        delete cleaned[field];
      });
      return cleaned;
    }
    return v;
  }
}
```

## 遇到的问题及解决方案

### 问题 1: Jest 模块解析失败

**症状**: `Cannot find module '../models/AuditLog'`

**原因**: Jest 的模块解析路径与 Node.js 不同

**解决**:
1. 配置 `moduleDirectories` 和 `moduleNameMapper`
2. 使用 `path.join(__dirname, '../../models/AuditLog')` 绝对路径导入

### 问题 2: 测试内存溢出

**症状**: JavaScript heap out of memory

**原因**: MongoDB Memory Server 占用大量内存，加上 coverage 收集导致内存不足

**解决**:
1. 禁用 coverage 收集 (`collectCoverage: false`)
2. 设置合理的超时时间 (`testTimeout: 30000`)
3. 使用 `timeout` 命令限制测试运行时间

### 问题 3: 索引重复创建

**风险**: 多次运行迁移脚本可能导致索引冲突

**解决**:
1. 检查索引是否已存在
2. 只在索引不完整时重新创建
3. 提供详细的日志输出
