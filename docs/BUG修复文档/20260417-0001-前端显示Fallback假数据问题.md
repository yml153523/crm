## 3. 根因分析

### 3.1 问题定位过程

#### 第一层诊断：API返回空数据 ⚠️

运行自动化诊断脚本 [diagnose_crm_system.py](scripts/diagnose_crm_system.py)：

```
❌ API [课程列表]: HTTP 200, 但数据为空 (0条)
❌ API [视频列表]: HTTP 200, 但数据为空 (0条)
⚠️ API 返回空数据 → 前端触发 Fallback 机制
```

#### 第二层诊断：数据库为空 🔴

通过SSH连接服务器检查：

```bash
$ mongo crm_system --eval "db.courses.countDocuments()"
# 结果: 0  ← 数据库中没有课程数据！
```

#### 第三层诊断：**错误的数据库** 🎯 **根因确认！**

检查后端配置文件 `/root/crm/server/.env`：

```env
PORT=5011
MONGODB_URI=mongodb://localhost:27017/crm_db  # ← 实际使用的数据库名称
JWT_SECRET=crm-secret-key-2026
```

**关键发现：**
- ❌ 种子数据导入到: `crm_system` (错误)
- ✅ 后端实际查询: `crm_db` (正确)
- → **数据库名称不匹配导致API返回空数据！**

### 8.1 本次问题的教训

#### 教训1: **数据库名称必须与配置保持一致** 🔴

**错误做法：**
```python
# 硬编码错误的数据库名称
DB_NAME = "crm_system"  # ← 凭感觉写的名字
```

**正确做法：**
```python
# 从配置文件或环境变量读取
import os
DB_NAME = os.getenv("MONGODB_URI", "").split("/")[-1] or "crm_db"
# 或者直接读取 .env 文件
```

#### 教训2: **部署前必须验证数据存在性** 🟡

应该在部署脚本中加入数据验证步骤：
