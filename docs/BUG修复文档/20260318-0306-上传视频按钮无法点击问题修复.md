# BUG 修复文档

## 基本信息
- **修复日期**: 2026-03-18
- **问题标题**: 上传视频按钮无法点击/上传失败
- **严重级别**: 高（阻断核心功能）
- **修复状态**: ✅ 已完成

## 问题描述

### 现象
1. 用户在上传视频页面点击"上传视频"按钮后无响应或报错
2. 错误信息：`上传失败：Unexpected token '<', '<html><h...' is not valid JSON`
3. 浏览器控制台显示：`POST http://192.168.66.3:8081/api/videos/upload 500 (Internal Server Error)`
4. 后端日志显示：`connect ECONNREFUSED 127.0.0.1:27017`（MongoDB 连接失败）

### 期望结果
- 点击上传按钮后，视频应该成功上传到服务器
- 上传成功后显示成功提示
- 自动创建上课提醒（如果提供了课程信息）

## 问题根因分析

### 四层问题

1. **API 地址配置问题**
   - 前端代码使用相对路径 `/api`，依赖 Nginx 代理
   - 但前端构建产物部署目录与 Nginx 配置不一致
   - Nginx 配置：`/home/work/crm/dist-admin`
   - 实际构建目录：`/usr/src/work/crm/dist-admin`

2. **启动脚本部署路径问题**
   - `start_services.sh` 部署到 `/var/www/crm-admin`
   - Nginx 配置使用 `/home/work/crm/dist-admin`
   - 导致 Nginx 提供的是旧版本的构建文件

3. **环境变量配置问题**
   - `.env` 文件配置了硬编码的 API 地址
   - 影响构建时的配置

4. **MongoDB 服务崩溃**
   - MongoDB 服务异常退出（core-dump）
   - 导致后端无法连接数据库
   - 所有需要数据库的操作都失败

## 修复方案

### 1. 修复 API 配置 (`src/utils/api.js`)

**修改前**:
```javascript
// 智能配置 API 地址
const getApiBaseUrl = () => {
  if (process.env.EXPO_PUBLIC_API_URL) {
    return process.env.EXPO_PUBLIC_API_URL;
  }
  // ...
};
```

**修改后**:
```javascript
// Web 平台使用相对路径，通过 Nginx 代理到后端
// 非 Web 平台使用本地地址
export const API_BASE_URL = (typeof window !== 'undefined' && window.location)
  ? '/api'
  : 'http://localhost:5011/api';
```

### 2. 修复启动脚本部署路径 (`scripts/start_services.sh`)

**新增部署目录变量**:
```bash
# Also deploy to nginx configured directories
NGINX_WWW_ADMIN="/home/work/crm/dist-admin"
NGINX_WWW_USER="/home/work/crm/dist-user"
```

**新增部署函数**:
```bash
deploy_nginx_dirs() {
  echo "[start_services] deploying to nginx configured directories..."
  deploy_static "$DIST_ADMIN" "$NGINX_WWW_ADMIN"
  deploy_static "$DIST_USER" "$NGINX_WWW_USER"
}
```

**在 main 函数中调用**:
```bash
main() {
  build_frontend
  deploy_static "$DIST_ADMIN" "$WWW_ADMIN"
  deploy_static "$DIST_USER" "$WWW_USER"
  
  # 新增：部署到 Nginx 配置目录
  deploy_nginx_dirs
  
  install_nginx_conf
  # ...
}
```

### 3. 删除环境变量文件

删除 `.env` 文件，避免硬编码配置影响构建。

### 4. 重启 MongoDB 和后端服务

```bash
# 重启 MongoDB
sudo systemctl restart mongod

# 重启后端服务
sudo pm2 restart crm-backend
```
