## 3. 根因分析

### 3.1 排查过程

#### 第一步：检查HTML文件
```bash
$ cat /var/www/crm-uniapp/admin/index.html
# 发现引用: /assets/index-DFM_b0YZ.js?v=20260414233420
```

#### 第二步：检查JS文件是否存在
```bash
$ ls /var/www/crm-uniapp/admin/assets/index-DFM_b0YZ.js
# 结果: 文件不存在！
```

#### 第三步：对比构建产物
```bash
$ ls /home/liuyeming/work/crm/dist-admin/assets/ | grep "index-DFM"
# 结果: index-DFM_b0YZ.js?v=20260414233420 (注意: 文件名包含 ?v=...)
```

#### 第四步：定位配置错误
```bash
$ grep -A3 "entryFileNames" vite.config.ts
# 发现:
entryFileNames: `assets/[name]-[hash].js?v=${timestamp}`,
```

### 3.2 根本原因

❌ **错误的Vite配置导致文件名异常**

**问题代码** ([vite.config.ts](../../vite.config.ts)):
```typescript
// ❌ 错误配置 (已修复)
build: {
  rollupOptions: {
    output: {
      entryFileNames: `assets/[name]-[hash].js?v=${timestamp}`,
      //                  ↑ 这里的 ?v=... 被当作文件名的一部分！
    }
  }
}
```

**影响链路**:
```
vite.config.ts 配置错误
    ↓
构建时生成文件名: index-DFM_b0YZ.js?v=20260414233420
    ↓
index.html引用: <script src="/assets/index-DFM_b0YZ.js?v=20260414233420">
    ↓
浏览器解析URL: 文件名 = "index-DFM_b0YZ.js", 参数 = "?v=20260414233420"
    ↓
服务器查找: /assets/index-DFM_b0YZ.js → ❌ 文件不存在!
    ↓
JS加载失败 → 页面白屏
```

### 3.3 技术细节

**为什么会出现这个问题？**

1. **Vite的rollupOptions.output配置**用于控制构建输出的文件名模板
2. 模板中的 `?v=${timestamp}` 被字面量替换到文件名中
3. 文件系统允许文件名包含 `?` 字符（虽然不推荐）
4. 但浏览器解析URL时，`?` 后面的内容被当作**查询参数**而非文件名
5. 导致服务器找不到实际文件

**正确的做法**:
- 文件名应该保持干净（不含特殊字符）
- 版本号应该通过**URL查询参数**添加（在HTML引用时）
- 或者在Nginx层控制缓存策略

## 4. 修复方案

### 4.1 修复内容

**修改文件**: [vite.config.ts](../../vite.config.ts)

**修改前** (❌ 错误):
```typescript
import { defineConfig } from 'vite'
import uni from '@dcloudio/vite-plugin-uni'

const timestamp = new Date().toISOString().slice(0, 19).replace(/[-:T]/g, '')

export default defineConfig({
  plugins: [uni()],
  build: {
    rollupOptions: {
      output: {
        entryFileNames: `assets/[name]-[hash].js?v=${timestamp}`,  // ❌ 错误
        chunkFileNames: `assets/[name]-[hash].js?v=${timestamp}`,   // ❌ 错误
        assetFileNames: `assets/[name]-[hash].[ext]?v=${timestamp}`  // ❌ 错误
      }
    },
  },
  server: { ... }
})
```

**修改后** (✅ 正确):
```typescript
import { defineConfig } from 'vite'
import uni from '@dcloudio/vite-plugin-uni'

export default defineConfig({
  plugins: [uni()],
  // 移除了错误的 build.rollupOptions 配置
  // 使用 Vite 默认的文件命名规则
  server: {
    port: 5173,
    proxy: {
      '/api': {
        target: 'http://localhost:5011',
        changeOrigin: true,
        rewrite: (path) => path.replace(/^\/api/, '')
      }
    }
  }
})
```

## 8. 经验教训

### 8.1 技术教训

1. **⚠️ Vite配置需谨慎**
   - `rollupOptions.output` 的文件名模板会直接影响最终输出
   - 避免在文件名中使用URL保留字符（? & # 等）
   - 自定义配置前必须充分测试

2. **✅ 版本号控制的正确姿势**
   - **推荐方式1**: Nginx层控制（Cache-Control头）
   - **推荐方式2**: HTML引用时加查询参数（如 `?v=xxx`）
   - **禁止方式**: 将查询参数嵌入文件名本身

3. **🔍 部署后必做验证**
   - 不能假设构建成功=部署成功
   - 必须验证关键资源文件的**实际可访问性**
   - 建议自动化验证替代人工检查

### 8.2 流程教训

1. **构建→部署→验证 应该是原子操作**
   - 本次错误：修改了vite.config.ts后未充分测试就部署
   - 改进：应该在本地先验证构建产物，再推送到服务器

2. **快速回滚机制很重要**
   - 如果发现问题，应立即回滚到上一个可用版本
   - 建议保留最近3个版本的备份
