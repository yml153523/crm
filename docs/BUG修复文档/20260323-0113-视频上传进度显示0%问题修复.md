# BUG 修复报告

## 1. 问题概述

### 问题描述
- **发生时间**: 2026-03-23
- **问题类型**: 前端UI问题
- **严重级别**: 中

### 问题现象
上传视频时，进度条一直显示0%，无法看到真实的上传进度，用户体验较差。

### 影响范围
- **页面**: 管理端 -> 课库 -> 上传视频页面
- **功能**: 普通视频上传（非分片上传）
- **影响**: 无法看到上传进度，用户不知道上传是否在进行

## 2. 问题定位

### 复现步骤
1. 登录管理端
2. 进入"课库"页面
3. 点击上传视频
4. 选择视频文件并填写标题
5. 点击上传按钮
6. 观察进度条 - 一直显示0%

### 根因分析

在 `src/screens/UploadVideoScreen.js` 的 `handleUpload` 函数中：

**问题代码**（修复前）：
```javascript
const response = await fetch(`${API_BASE_URL}/videos/upload`, {
  method: 'POST',
  headers: token ? {
    'Authorization': 'Bearer ' + token
  } : {},
  body: formData,
});
```

**根本原因**：
- 使用了 `fetch` API 进行文件上传
- `fetch` API 不支持上传进度监听
- 只有分片上传（`handleChunkUpload`）有进度更新
- 普通上传没有进度更新机制

## 3. 修复方案

### 修复思路
1. 在 Web 平台使用 `XMLHttpRequest` 替代 `fetch`
2. 利用 `xhr.upload.addEventListener('progress')` 监听上传进度
3. 保持非 Web 平台（移动设备）继续使用 `fetch`
4. 在进度事件中调用 `setProgress` 更新进度条

### 修改内容

**文件**: `src/screens/UploadVideoScreen.js`

**关键修改**：
```javascript
if (Platform.OS === 'web') {
  await new Promise((resolve, reject) => {
    const xhr = new XMLHttpRequest();
    
    // 监听上传进度
    xhr.upload.addEventListener('progress', (e) => {
      if (e.lengthComputable) {
        const percentComplete = e.loaded / e.total;
        console.log('上传进度:', percentComplete);
        setProgress(percentComplete);
      }
    });
    
    xhr.addEventListener('load', () => {
      // 处理响应
    });
    
    xhr.open('POST', `${API_BASE_URL}/videos/upload`);
    
    if (token) {
      xhr.setRequestHeader('Authorization', 'Bearer ' + token);
    }
    
    xhr.send(formData);
  });
} else {
  // 非 Web 平台继续使用 fetch
  const response = await fetch(...);
}
```

### 修改文件清单
- `src/screens/UploadVideoScreen.js` - 修改 `handleUpload` 函数
