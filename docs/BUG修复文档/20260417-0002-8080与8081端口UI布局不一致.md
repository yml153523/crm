### 3.2 根因确认 🎯

**根本原因：部署时间不一致导致的潜在差异**

| 因素 | 8080端口 | 8081端口 | 差异 |
|------|---------|---------|------|
| 部署时间 | 23:39 | 23:27 | **12分钟** |
| 构建来源 | dist-user (较新) | dist-user (较早) | 可能版本不同 |
| 数据来源 | 同一构建产物？ | 同一构建产物？ | ❓ 不确定 |

**可能的具体原因：**

1. **构建产物版本不同**: 两次部署之间可能有代码修改
2. **浏览器缓存差异**: 不同时间的部署导致缓存了不同版本
3. **动态内容加载**: API返回数据的时间差导致渲染差异
4. **文件复制不完整**: 某次部署可能遗漏了部分资源

## 8. 经验教训与改进建议

### 8.1 本次问题的教训

#### 教训1: **多端口部署必须保持同步** 🔴

**错误做法：**
```bash
# 分别在不同时间部署到不同端口
npm run build:user && deploy_to_8080.sh  # 时间A
# ... 12分钟后 ...
npm run build:user && deploy_to_8081.sh  # 时间B
```

**正确做法：**
```bash
# 统一构建，同时部署到多个端口
npm run build:user
deploy_to_all_ports.sh  # 同时部署到8080和8081
```

#### 教训2: **建立部署版本管理机制** 🟡

建议实现：

```yaml
# deployment-manifest.yml
version: "20260417.0022"
build_time: "2026-04-17T00:22:00+08:00"
ports:
  - port: 8080
    path: /var/www/crm-admin
    deployed_at: "2026-04-17T00:22:45"
    checksum: "f9fdb0e9b1..."
    
  - port: 8081
    path: /var/www/crm-user
    deployed_at: "2026-04-17T00:22:45"
    checksum: "f9fdb0e9b1..."  # 必须相同！
```

#### 教训3: **部署后自动验证一致性** 🟢

添加到CI/CD流程：

```python
def verify_deployment_consistency():
    """部署完成后自动验证多端口一致性"""
    ports = [8080, 8081]
    checksums = {}
    
    for port in ports:
        checksum = get_file_checksum(f"/var/www/crm-{get_dir_name(port)}/assets/*.js")
        checksums[port] = checksum
    
    if len(set(checksums.values())) > 1:
        send_alert("⚠️ 端口间存在不一致！")
        return False
    
    return True
```
