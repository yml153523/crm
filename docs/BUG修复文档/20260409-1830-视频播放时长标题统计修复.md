## 二、根因分析

### 2.1 后端问题 (`server/routes/video.js`)

**BUG #1 & #3 - 无真实时长获取**
```javascript
// ❌ 原代码：没有duration字段
const video = await Video.create({
  title: title || req.file.originalname,
  // duration 字段缺失 → 默认为0
})
```

**BUG #2 - 标题处理不当**
```javascript
// ❌ 原代码：空字符串title会fallback到originalname
title: title || req.file.originalname  // '' 是falsy！
```

### 2.2 前端问题 (`src/pages/admin/video/list.vue`)

**BUG #3 - 错误的时长估算算法**
```javascript
// ❌ 原代码：用文件大小粗略估算（7MB × 5 = 35秒）
function formatDurationFromSeconds(sizeBytes) {
  return Math.floor(sizeBytes / (1024*1024) * 5)  // 完全不准！
}
// 然后被formatDuration再次转换，导致 35秒 → 894秒 → "14分49秒"
```

**BUG #4 - fallback到硬编码假数据**
```javascript
// ❌ 原代码：API失败时使用getMockData()
catch (error) {
  videoList.value = getMockData()  // 返回4条假数据！
}
```

## 三、修复方案

### 3.1 后端修复 ✅

**文件**: `server/routes/video.js`

#### 修改1: 新增ffprobe获取真实时长
```javascript
function getVideoDuration(filePath) {
  return new Promise((resolve) => {
    exec(`ffprobe -v error -show_entries format=duration ...`, (err, stdout) => {
      if (err) {
        // 降级：按文件大小估算 (1MB ≈ 8s for 1080p)
        const stats = fs.statSync(filePath)
        resolve(Math.round(stats.size / (1024*1024) * 8))
      } else {
        resolve(Math.round(parseFloat(stdout.trim())))  // 真实秒数
      }
    })
  })
}
```

#### 修改2: 上传时保存真实时长 + 处理标题
```javascript
let videoTitle = (title && title.trim()) ? title.trim() : req.file.originalname
if (videoTitle.includes('.mp4')) videoTitle = videoTitle.replace(/\.[^.]+$/, '')

const duration = await getVideoDuration(req.file.path)

const video = await Video.create({
  title: videoTitle,
  duration: duration,  // ← 真实时长（秒）
  // ...
})
```

### 3.2 前端修复 ✅

**文件**: `src/pages/admin/video/list.vue`

#### 修改1: 重写formatDuration函数
```javascript
function formatDuration(duration): string {
  let totalSeconds = typeof duration === 'number' ? Math.round(duration) : parseString(duration)
  
  if (totalSeconds >= 3600) return `${hours}时${mins}分`
  if (totalSeconds >= 60) return `${mins}分${secs}秒`
  return `${totalSeconds}秒`
}
```

#### 修改2: 移除错误估算函数
- 删除 `formatDurationFromSeconds()` 函数
- 直接使用后端返回的 `v.duration`（数字类型）

#### 修改3: 移除假数据fallback
- API失败时返回空数组 `[]`，不再调用 `getMockData()`

## 五、回归风险

| 风险项 | 影响 | 缓解措施 |
|--------|------|----------|
| ffprobe未安装 | 上传时降级为文件大小估算 | 已做降级处理；建议服务器安装ffprobe |
| 大文件上传超时 | >100MB视频可能失败 | Nginx已配置client_max_body_size 500M和300s超时 |
| 浏览器编码支持 | 部分旧浏览器可能不支持H.265 | 建议上传H.264编码的MP4 |
