## 🚨 Phase 2: 运行时 BUG 修复 (5个)

### BUG #7: 创建红包缺少必填字段
**文件**: `server/routes/admin-red-packets.js` (POST /)  
**错误信息**: 
```json
{
  "success": false,
  "message": "RedPacket validation failed",
  "error": {
    "type": "VALIDATION_ERROR",
    "details": [
      { "field": "taskType", "message": "Path `taskType` is required" }
    ]
  }
}
```
**根因**: RedPacket模型要求taskType字段必填，但API未提供  
**影响范围**: 无法创建任何红包（阻断性问题）  
**修复方案**: 在创建时添加默认值:
```javascript
{
  taskType: 'manual',
  taskConfig: {},
  minAmount: 1,
}
```
**验证**: 成功创建红包，返回完整文档ID `69dc34e12d8fe11879e84481`

### BUG #8: 路由顺序冲突导致列表查询失败
**文件**: `server/app.js` (setupRoutes方法)  
**错误信息**: 
```json
{
  "success": false,
  "message": "Cast to ObjectId failed for value \"list\""
}
```
**根因**: Express按顺序匹配路由，`/:id` 在 `/list` 之前注册，导致"list"被当作ID  
**影响范围**: 列表、统计、导出等所有子路由全部失败（严重问题）  
**修复方案**: 调整路由挂载顺序，将具体路径放在通配路径之前:
```javascript
// 正确顺序
app.use('/api/admin/red-packets/list', ...)     // 具体路径优先
app.use('/api/admin/red-packets/stats', ...)
app.use('/api/admin/red-packets/export', ...)
app.use('/api/user/red-packets', ...)
app.use('/api/admin/red-packets', ...)             // 通配路径最后
```
**验证**: 列表查询返回正确数据，总数=2，分页正常

### BUG #9: ExportTask模型createdBy字段必填限制
**文件**: `server/models/ExportTask.js` (L11-15)  
**错误信息**: 
```json
{
  "success": false,
  "message": "创建导出任务失败"
}
```
**服务器日志**: 
```
ValidatorError: Path `createdBy` is required.
ExportTask validation failed
```
**根因**: createdBy字段设置为required:true，但未登录时无法提供有效ObjectId  
**影响范围**: 导出功能完全不可用  
**修复方案**: 将字段改为可选并设置默认值:
```javascript
createdBy: {
  type: mongoose.Schema.Types.ObjectId,
  ref: 'User',
  required: false,
  default: null
}
```
**验证**: 导出任务创建成功，taskId=`export_1776039237637_cwm6e526x`
