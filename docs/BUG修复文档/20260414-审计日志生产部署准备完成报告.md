## 五、使用指南

### 5.1 如何获取Token

```bash
# 步骤1: 登录获取Token
curl -X POST http://localhost:3001/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"phone":"admin","password":"admin123"}'

# 响应示例:
# {
#   "success": true,
#   "data": {
#     "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
#     "user": { ... }
#   }
# }

# 步骤2: 保存Token到变量
TOKEN="eyJhbGciOiJIUzI1NiIs..."
```

### 5.2 不同角色的操作示例

#### 普通用户 (role: 'user')

```bash
# ✅ 允许的操作
curl -H "Authorization: Bearer $TOKEN" \
  http://localhost:3001/api/audit-logs

curl -H "Authorization: Bearer $TOKEN" \
  http://localhost:3001/api/audit-logs/statistics/overview

# ❌ 被拒绝的操作 (返回403)
curl -X POST -H "Authorization: Bearer $TOKEN" \
  http://localhost:3001/api/audit-logs/export

curl -X DELETE -H "Authorization: Bearer $TOKEN" \
  http://localhost:3001/api/audit-logs/cleanup
```

#### 管理员 (role: 'admin')

```bash
# ✅ 允许的所有操作（除清理外）
curl -X POST -H "Authorization: Bearer $ADMIN_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{}' \
  http://localhost:3001/api/audit-logs/export \
  -o audit-logs.csv

# ❌ 清理仍被拒绝 (需要super_admin)
curl -X DELETE -H "Authorization: Bearer $ADMIN_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"beforeDate":"2020-01-01"}' \
  http://localhost:3001/api/audit-logs/cleanup
# 返回: 403 FORBIDDEN - 需要超级管理员权限
```

#### 超级管理员 (role: 'super_admin')

```bash
# ✅ 全部操作允许
curl -X DELETE -H "Authorization: Bearer $SUPER_ADMIN_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"beforeDate":"2020-01-01"}' \
  http://localhost:3001/api/audit-logs/cleanup

# 返回: 200 OK - { deletedCount: N }
```

### 5.3 错误码速查表

| HTTP状态码 | 错误码 | 含义 | 解决方案 |
|-----------|--------|------|----------|
| 401 | UNAUTHORIZED | 缺少认证令牌 | 在Header中添加 Authorization: Bearer {token} |
| 401 | TOKEN_EXPIRED | 令牌已过期 | 重新登录获取新Token |
| 401 | INVALID_TOKEN | 令牌无效 | 检查Token是否完整/篡改 |
| 403 | FORBIDDEN | 权限不足 | 联系管理员提升角色 |
| 404 | NOT_FOUND | 资源不存在 | 检查ID是否正确 |
