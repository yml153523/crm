## 3. 根因分析

### 问题定位过程

#### 第一层现象：白屏
- 初步怀疑：JS 编译错误、Vue 组件异常、权限逻辑 bug

#### 第二层排查：检查构建产物
```bash
# 检查 dist-admin 目录
ls -lh dist-admin/assets/*.js
# 结果：文件存在，大小正常（9.4KB）
```

#### 第三层排查：检查服务器部署文件
```bash
cat /var/www/crm-uniapp/admin/index.html
```

**🎯 发现关键错误：**
```html
<!-- ❌ 错误的 HTML（sed 命令生成） -->
<script type="module" crossorigin src="src="/assets/index-DZKnLwo7.js"?v=20260415091651"></script>

<!-- ✅ 正确的 HTML -->
<script type="module" crossorigin src="/assets/index-DZKnLwo7.js?v=20260415091651"></script>
```

### 根本原因

**sed 命令中的 `&` 符号使用错误**

执行的命令：
```bash
sed -i "s|src=\"/assets/[^\"]*\.js\"|src=\"&?v=$TIMESTAMP\"|g" index.html
```

**执行流程分析：**
1. **正则匹配**: `src="/assets/[^\"]*\.js"` 
   - 匹配到: `src="/assets/index-DZKnLwo7.js"`
   
2. **替换操作**: `src="&?v=timestamp"`
   - `&` 在 sed 中代表**整个匹配内容**
   - 替换结果: `src="` + `src="/assets/index-DZKnLwo7.js"` + `?v=timestamp"`
   - 最终生成: `src="src="/assets/index-DZKnLwo7.js"?v=timestamp"`

3. **浏览器解析失败**
   - HTML 解析器遇到重复的 `src` 属性
   - 第二个 `src=` 被视为未知属性
   - `?v=...` 在引号外，被视为标签文本
   - **结果：script 标签无效 → JS 未加载 → 白屏**

### 影响范围评估

| 影响项 | 说明 |
|--------|------|
| 用户影响 | 所有访问 8081 端口的用户无法使用系统 |
| 功能影响 | 登录、Dashboard、所有管理功能不可用 |
| 数据影响 | 无（仅前端展示问题） |
| 时间影响 | 从 09:16 部署后持续约 15 分钟 |

## 4. 修复方案

### 4.1 紧急修复（已执行）

```bash
# 步骤 1：修复重复的 src 属性
sudo sed -i 's|src="src="/assets/|src="/assets/|g' /var/www/crm-uniapp/admin/index.html

# 步骤 2：修复 ?v= 参数位置（移入引号内）
sudo sed -i 's|\.js"?v=|.js?v=|g' /var/www/crm-uniapp/admin/index.html

# 步骤 3：验证修复结果
cat /var/www/crm-uniapp/admin/index.html | grep 'script'
# 期望输出：<script type="module" crossorigin src="/assets/index-DZKnLwo7.js?v=20260415091651"></script>
```

### 4.2 正确的版本号添加方式（推荐）

#### 方案 A：使用正确的 sed 命令
```bash
# ✅ 正确：不使用 &，直接在匹配组内添加参数
sudo sed -i 's|src="/assets/\([^"]*\.js\)"|src="/assets/\1?v='"$TIMESTAMP"'"|g' index.html
```

#### 方案 B：使用 awk（更安全）
```bash
awk -v ts="$TIMESTAMP" '/src="\/assets\/.*\.js"/ {
  match($0, /src="[^"]+"/)
  old = substr($0, RSTART, RLENGTH)
  gsub(/\./, ".", old)  # 保持原样
  sub(/\.js"/, ".js?v=\""ts"\"", old)
  sub(/src="[^"]+/, old)
} {print}' index.html > index.html.tmp && mv index.html.tmp index.html
```

#### 方案 C：直接修改 Vite 配置（最佳实践）
```javascript
// vite.config.ts
export default defineConfig({
  build: {
    rollupOptions: {
      output: {
        entryFileNames: `assets/[name]-[hash].js?v=${Date.now()}`
      }
    }
  }
})
```
⚠️ **注意**：之前尝试此方案时曾导致白屏（文件名包含 `?` 号），需要确保 Nginx 配置正确处理查询参数。

## 7. 经验教训与预防措施

### 7.1 技术教训

1. **sed 命令中的特殊字符**
   - `&` 代表整个匹配内容，容易导致重复
   - 替换字符串中需要转义的特殊字符：`\`, `"`, `/`, `&`
   
2. **HTML 语法容错性差**
   - 浏览器对 HTML 错误的容忍度较低
   - script 标签错误会导致整个应用无法加载
   
3. **部署自动化需要验证**
   - 每次部署后应自动验证关键文件的正确性
   - 特别注意正则表达式替换的结果
