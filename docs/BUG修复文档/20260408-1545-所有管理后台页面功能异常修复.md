### 根因分析

**根本原因**: 所有页面使用了已移除的 **uView Plus** 组件库组件：
- `<u-button>` - 按钮组件 ❌ 不渲染
- `<u-icon>` - 图标组件 ❌ 不渲染  
- `<u-tag>` - 标签组件 ❌ 不渲染
- `<u-empty>` - 空状态组件 ❌ 不渲染
- `<u-tabs>` - 标签页组件 ❌ 不渲染
- `<u-loadmore>` - 加载更多组件 ❌ 不渲染

**技术背景**:
- 项目初期引入 uView Plus UI 框架
- 因 SCSS 编译错误和导入失败，uView Plus 已从项目完全移除 ([main.ts](file:///home/liuyeming/work/crm/src/main.ts))
- 但各业务页面仍引用这些不存在的组件，导致 Vue 无法识别并渲染它们
- 结果：所有使用 uView Plus 的元素在 DOM 中完全不可见

### 技术改进点

#### 事件绑定修正
```diff
- @click="handleClick"
+ @tap="handleTap"
```
**原因**: UniApp H5环境必须使用 `@tap` 替代 `@click` 才能正确触发事件

#### 样式单位统一
```diff
- padding: 16rpx;  // rpx 单位（小程序专用）
+ padding: 16px;   // px 单位（H5兼容）
```
**原因**: H5端对 rpx 支持不稳定，统一使用 px 保证跨平台一致性

#### 组件替换映射表
| uView Plus 组件 | 替代方案 | 效果 |
|----------------|---------|------|
| `<u-button type="primary">` | `<view class="btn btn-primary">` | ✅ 完全可控样式 |
| `<u-icon name="xxx">` | `<text>emoji</text>` | ✅ 无依赖 |
| `<u-tag text="xxx">` | `<view class="tag">` | ✅ 自定义样式 |
| `<u-empty text="xxx">` | `<view class="empty-state">` | ✅ 自定义空状态 |
| `<u-tabs :list="tabs">` | `<view class="tab-item">` | ✅ 手动控制 |
| `<u-loadmore>` | `<view class="load-more">` | ✅ 简单文本 |
