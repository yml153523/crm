## 六、发现的问题及解决方案

### 问题 1: 服务器启动方式不一致

**严重程度**: 低 (信息性)  
**发现时间**: 测试初期  
**问题描述**:

直接运行 `node app.js` 与使用 `app.initialize().then(() => app.start())` 行为不一致。

**根本原因**:

`RedPacketApp` 类需要先调用 `initialize()` 方法完成数据库连接、中间件配置等初始化工作，然后才能调用 `start()` 启动HTTP服务器。如果跳过 initialize() 直接调用 start()，会导致 this.server 为 null。

**影响范围**:
- 开发调试时可能遇到启动失败
- 生产环境通常通过正确的启动脚本启动，不受影响

**解决方案**:

已在测试过程中掌握正确的启动方式：

```javascript
const redPacketApp = require('./app');
redPacketApp.initialize()
  .then(() => redPacketApp.start(3001))
  .then(() => console.log('✅ 服务器已启动'))
  .catch(err => console.error('❌ 启动失败:', err.message));
```

**建议改进**:

可以在 `package.json` 的 start 脚本中固化此流程，或在 app.js 底部添加自动初始化逻辑。

**状态**: ✅ 已解决（不影响功能）
