## 2. 根因分析

### 2.1 问题定位

#### ❌ **问题1: 页面路由配置错误**
**文件**: [src/pages.admin.json](src/pages.admin.json) (修改前)

```json
{
  "pages": [
    { "path": "pages/index/index" },  // ← 错误！首页作为第一页
    // ...
  ]
}
```

**根因**: 管理端配置将通用首页 (`pages/index/index`) 设为第一页，导致访问时显示选择界面而非登录页。

#### ❌ **问题2: 登录表单字段错误**
**文件**: [src/pages/admin/login.vue](src/pages/admin/login.vue) (修改前)

```vue
<input v-model="phone" type="number" placeholder="请输入管理员手机号" />
```

**根因**: 
- 字段名使用 `phone`（手机号），应改为 `username`（用户名）
- placeholder 提示文字错误
- 未预填默认账号密码

#### ❌ **问题3: 按钮事件绑定问题**
**文件**: [src/pages/admin/login.vue](src/pages/admin/login.vue) (修改前)

```vue
<button @click="handleLogin">  <!-- H5端可能不兼容 -->
```

**根因**: 在 UniApp H5 端，部分情况下 `@click` 可能不如 `@tap` 可靠。

**效果**: 访问 8080 端口时直接显示管理员登录页面。

#### ✅ **修改2: 管理后台登录页面重写**

**文件**: [src/pages/admin/login.vue](src/pages/admin/login.vue)

**关键改动**:
```vue
<!-- 修改前 -->
<input v-model="phone" type="number" placeholder="请输入管理员手机号" />
<button @click="handleLogin">管理员登录</button>

<!-- 修改后 -->
<input v-model="username" type="text" placeholder="请输入管理员账号" />
<!-- 默认值 -->
const username = ref('admin')
const password = ref('admin123')
<!-- 事件绑定 -->
<button @tap="handleLogin">管理员登录</button>
```

**新增特性**:
- ✅ 预填默认账号: `admin`
- ✅ 预填默认密码: `admin123`
- ✅ 表单验证（用户名≥3位，密码≥6位）
- ✅ 演示模式支持（API不可用时自动进入）
- ✅ UI美化（渐变背景、阴影、动画）
