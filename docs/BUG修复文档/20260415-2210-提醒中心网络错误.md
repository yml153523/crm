## 3. 修复方案

### 修复策略
将所有 `fetch()` 调用替换为 `uni.request()` (UniApp官方推荐的HTTP请求方式)

### 具体修改

#### 修改1: loadRedPacketUsers() 函数 [第188-203行]

**修改前 (❌):**
```javascript
async function loadRedPacketUsers() {
  try {
    const res: any = await fetch('/api/remind/users/redPacket').then(r => r.json())
    if (res.success) {
      // ...处理数据
    }
  } catch (error) {
    console.error('加载红包用户失败:', error)
    redPacketUsers.value = getMockRedPacketUsers()
  }
}
```

**修改后 (✅):**
```javascript
async function loadRedPacketUsers() {
  try {
    const res: any = await new Promise((resolve, reject) => {
      uni.request({
        url: '/api/remind/users/redPacket',
        method: 'GET',
        header: {
          'Authorization': `Bearer ${uni.getStorageSync('token')}`  // 添加认证Token
        },
        success: (res) => resolve(res.data),
        fail: (err) => reject(err)
      })
    })
    
    if (res.success) {
      redPacketUsers.value = (res.data?.list || []).map((u: any) => ({
        ...u,
        phone: u.phone ? u.phone.replace(/(\d{3})\d{4}(\d{4})/, '$1****$2') : ''
      }))
      tabs.value[0].count = redPacketUsers.value.length
    }
  } catch (error) {
    console.error('加载红包用户失败:', error)
    redPacketUsers.value = getMockRedPacketUsers()
    tabs.value[0].count = 5
  }
}
```

#### 修改2: loadClassUsers() 函数 [第205-222行]

同样的替换逻辑：
- `fetch()` → `uni.request()`
- 添加 Authorization header
- 使用 Promise 包装以保持 async/await 语法

#### 修改3: sendBatchRemind() 函数 [第314-318行]

**修改前 (❌):**
```javascript
const result: any = await fetch('/api/remind/batch-send', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ userIds, type })
}).then(r => r.json())
```

**修改后 (✅):**
```javascript
const result: any = await new Promise((resolve, reject) => {
  uni.request({
    url: '/api/remind/batch-send',
    method: 'POST',
    header: {
      'Content-Type': 'application/json',
      'Authorization': `Bearer ${uni.getStorageSync('token')}`  // 添加Token
    },
    data: { userIds, type },  // uni.request使用data而非body
    success: (res) => resolve(res.data),
    fail: (err) => reject(err)
  })
})
```

### 关键改进点

| 改进项 | 修改前 | 修改后 |
|--------|--------|--------|
| HTTP客户端 | 原生 `fetch()` | UniApp `uni.request()` |
| 认证机制 | 无 | 携带 `Authorization: Bearer <token>` |
| 数据格式 | `body: JSON.stringify()` | `data: {}` (自动序列化) |
| 路径解析 | 相对路径可能失效 | 通过BASE_URL正确解析 |
| 错误处理 | 抛出原始错误 | 统一错误处理流程 |

## 6. 经验总结

### 技术教训

1. **UniApp H5模式的限制**
   - ❌ 不要使用原生 `fetch()` / `XMLHttpRequest` / `axios` 等Web API
   - ✅ 必须使用 `uni.request()` 进行所有HTTP请求
   - 原因: UniApp会对请求进行统一封装、拦截、错误处理

2. **认证Token的重要性**
   - 后端大多数API需要JWT Token认证
   - 前端每次请求必须携带: `Authorization: Bearer <token>`
   - 即使是GET请求也需要认证（除了公开接口如健康检查）

3. **代码一致性原则**
   - 同一项目中应统一使用一种HTTP客户端
   - 本项目已封装了 [`@/api/request.ts`](file:///home/liuyeming/work/crm/src/api/request.ts) 模块
   - 所有页面都应该通过这个模块发起请求，而不是直接调用底层API

4. **Mock数据的陷阱**
   - 当前代码在catch块中使用Mock数据作为降级方案
   - 这可能导致开发时发现问题延迟到生产环境
   - 建议: 开发阶段应该让错误更明显，而不是静默降级
