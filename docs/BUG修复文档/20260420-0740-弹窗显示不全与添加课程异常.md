## 2. 根因分析

### BUG1根因：移动端适配不足

**问题代码位置**: [course/library.vue](file:///home/liuyeming/work/crm/src/pages/admin/course/library.vue) 第867-904行

**具体问题**：

```css
/* 修改前的问题样式 */
.modal-overlay {
  padding: 16px;  /* ❌ 在小屏幕上占用过多空间 */
}

.modal-content {
  max-height: 85vh;  /* ❌ 固定高度，未考虑安全区域 */
}

.modal-body {
  max-height: 60vh;  /* ❌ 内容区域过小 */
  padding: 20px;     /* ❌ 内边距过大 */
}
```

**根本原因**：
1. **固定尺寸设计**：使用固定的vh单位，未考虑不同手机的屏幕尺寸差异
2. **缺少安全区域适配**：未处理刘海屏/底部指示条的安全区域（safe-area-inset）
3. **内边距过大**：padding值在移动端显得臃肿
4. **滚动优化缺失**：缺少`-webkit-overflow-scrolling: touch`导致iOS滚动不流畅

### BUG2根因：前端请求处理逻辑缺陷

**问题代码位置**: [course/library.vue](file:///home/liuyeming/work/crm/src/pages/admin/course/library.vue) 第413-472行

**具体问题**：

```javascript
// 修改前的问题代码
async function submitForm() {
  // ... 表单验证 ...

  let res: any
  res = await uni.request({  /* ❌ 直接await，可能返回undefined */
    url: '/api/courses',
    method: 'POST',
    data: payload,
    header: { 'Authorization': `Bearer ${token}` }
    // ❌ 缺少 Content-Type 头
  })

  res = res as any  /* ❌ 强制类型转换，掩盖潜在错误 */

  if (res.statusCode === 200 || res.statusCode === 201) {  /* ❌ 未检查data.success */
```

3. **错误处理粗糙**：未区分网络错误、认证失败、业务错误等场景
4. **缺少加载状态**：用户点击后无反馈，不知道是否在处理中
5. **日志不足**：出错时难以定位问题

### 7.2 API请求封装建议（避免类似BUG2）

```javascript
// 建议创建统一的request工具函数
// utils/request.js
export function request(options) {
  return new Promise((resolve, reject) => {
    uni.request({
      url: options.url,
      method: options.method || 'GET',
      data: options.data,
      header: {
        'Content-Type': 'application/json',
        'Authorization': `Bearer ${uni.getStorageSync('token')}`,
        ...options.header
      },
      success: (res) => {
        if (res.statusCode === 401) {
          uni.navigateTo({ url: '/pages/login' })
          reject(new Error('Unauthorized'))
        } else {
          resolve(res.data)
        }
      },
      fail: (err) => reject(err)
    })
  })
}

// 使用示例
const result = await request({
  url: '/api/courses',
  method: 'POST',
  data: payload
})
```

**好处**：
- 统一处理认证、错误、loading等
- 减少重复代码
- 降低出错概率
