### 第二阶段：问题诊断与修复（关键！）

#### 问题1: POST /api/users 返回404
**根因分析**:
- `deploy_to_cloud.py` 脚本**只上传了前端静态文件**
- 未同步后端 server/ 目录的代码到云端
- 本地新增的路由文件（routes/user.js, routes/auditLog.js等）不存在于云端

**修复方案**:
1. 创建 `deploy_backend_to_cloud.py` - 上传核心server文件
2. 发现缺少18个路由文件中的大部分
3. 创建 `deploy_all_routes.py` - 补全所有路由文件
4. 发现还缺少14个models文件和controllers
5. 创建 `final_fix_deploy.py` - 上传所有依赖
6. 发现还缺少services/目录（6个服务层文件）
7. 创建 `complete_server_deploy.py` - 递归上传整个server目录
8. 最终使用 `deploy_services_fix.py` 完成所有services文件上传

**涉及文件统计**:
```
✅ 路由文件 (routes/): 18个
✅ 模型文件 (models/): 14个  
✅ 控制器文件 (controllers/): 4个
✅ 服务层文件 (services/): 6个
✅ 中间件文件 (middleware/): 3个
✅ 定时任务 (schedulers/): 1个
✅ 配置文件: server.js, package.json, .env等
总计: ~50个文件
```

#### 问题2: services/ 目录创建失败
**错误信息**: `[Errno 2] No such file` 或 `[Errno 13] Permission denied`
**根因**: Python的 `os.makedirs()` 只能操作本地文件系统，不能用于远程SFTP路径
**修复**: 改用 `ssh.exec_command("mkdir -p remote_path")` 在远程服务器创建目录

#### 问题3: 模块依赖链式缺失
**错误链**:
```
server.js → routes/auth.js → models/User.js, models/AuditLog.js
         → routes/admin-red-packets-export.js → services/ExportService.js
         → ... (更多嵌套依赖)
```
**解决方案**: 递归扫描并上传整个server目录（排除node_modules, tests）
