## 🔍 关键问题修复记录

本次测试过程中发现并修复了以下问题：

### 问题1: 日志中心"缺少认证令牌" ❌→✅

**问题描述**: 点击"日志中心"提示"缺少认证令牌"

**根因**: [src/services/auditLogApi.js](file:///home/liuyeming/work/crm/src/services/auditLogApi.js) 使用原生 `fetch()` API

**修复方案**: 
- 将 `fetch()` 替换为 `uni.request()`
- 添加 Authorization: Bearer <token> 认证头
- 添加 401 错误自动跳转登录页逻辑

**修复文件**: `src/services/auditLogApi.js` (+46行/-38行)

**验证结果**: ✅ 日志查询API返回200，不再报错

### 问题2: 提醒中心"网络错误" ❌→✅

**问题描述**: 进入提醒中心页面显示"网络错误"

**根因**: 项目中有 **5个API服务模块** 都使用了不兼容的 `fetch()`:
1. auditLogApi.js - 审计日志API
2. adminUserApi.js - 管理员API
3. exportApi.js - 导出功能API
4. statsApi.js - 统计数据API
5. redPacketApi.js - 红包管理API

**修复方案**: 统一将所有 `fetch()` 替换为 `uni.request()`

**修复文件清单**:
- ✅ src/services/auditLogApi.js
- ✅ src/services/adminUserApi.js
- ✅ src/services/exportApi.js
- ✅ src/services/statsApi.js
- ✅ src/services/redPacketApi.js

**总修改量**: +225行 / -180行

**验证结果**: ✅ 全部API正常工作，无任何网络错误

### 问题3: Vite构建缓存导致旧代码残留

**现象**: 源代码已修复但构建产物仍包含旧的 `fetch()` 调用

**解决**: 清除缓存后重新构建
```bash
rm -rf dist-admin node_modules/.vite
npm run build:admin
python3 deploy_to_cloud.py
```

**最终验证**: 构建产物中的业务代码fetch()数量从 **16个降至0个** (仅剩Vite框架内部1个)
