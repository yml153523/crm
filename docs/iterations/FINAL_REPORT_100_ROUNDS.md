## 🎯 支持的事件类型（12种）

### 内容管理类 (8种)

| 事件 | 触发场景 | 用户端行为 |
|------|---------|-----------|
| `video:created` | 管理员上传视频 | 刷新视频列表 + Toast提示 |
| `video:updated` | 编辑视频信息 | 刷新视频详情 |
| `video:deleted` | 删除视频 | 从列表移除 |
| `course:created` | 创建课程 | 刷新课程列表 + Toast提示 |
| `course:updated` | 更新课程 | 刷新课程详情 |
| `course:deleted` | 删除课程 | 从列表移除 |
| `product:created` | 添加商品 | 刷新商品列表 + Toast提示 |
| `product:updated` | 更新商品(含价格) | 刷新商品详情 |
| `product:deleted` | 下架商品 | 从列表移除 |

### 营销推广类 (4种)

| 事件 | 触发场景 | 用户端行为 |
|------|---------|-----------|
| `recommendation:published` | **发布推荐** | **高亮显示 + 特殊提示** |
| `redPacket:created` | 创建红包 | 刷新红包区域 |
| `redPacket:updated` | 编辑红包 | 刷新红包详情 |
| `redPacket:published` | **发布红包** | **弹窗提醒 + 强提示** |

## 💡 最佳实践

### 1. 管理端开发规范

```typescript
// ✅ 正确做法：始终使用Sync包装函数
const result = await courseSync.create(
  apiPost('/api/courses', data)
)

// ❌ 错误做法：直接调用API（不会触发实时同步）
const result = await apiPost('/api/courses', data)
```

### 2. 用户端开发规范

```typescript
// ✅ 正确做法：使用Composable自动管理生命周期
setup() {
  const { wsState } = useRealtimeSync({ autoConnect: true })
  const trigger = useDataRefresh('all')
  
  watch(trigger, () => {
    loadData()  // 自动刷新
  })
}

// ❌ 错误做法：手动管理连接（容易出错）
// let ws = new WebSocket(url)
// ws.onmessage = ... // 需要自己处理重连、错误等
```

### 3. 事件命名规范

```javascript
// 格式: {资源}:{动作}
// 资源: video, course, product, recommendation, redPacket, user, system
// 动作: created, updated, deleted, published

// ✅ 正确示例
'video:created'
'redPacket:published'
'system:notification'

// ❌ 错误示例
'addVideo'  // 不符合RESTful风格
'VIDEO_ADD'  // 不应该全大写
'createVideo'  // 顺序反了
```

## 🔒 安全性考虑

### 已实施的安全措施

✅ **Token认证**: WebSocket连接时可传递Token进行身份验证  
✅ **类型分离**: 管理端和用户端使用不同的连接类型(type=admin/user)  
✅ **消息格式校验**: 服务端解析前进行JSON格式验证  
✅ **CORS控制**: HTTP API层面已配置跨域策略  

### 建议增强（未来版本）

⚠️ **消息加密**: 使用WSS (WebSocket Secure) + TLS加密传输  
⚠️ **速率限制**: 防止恶意用户大量发送消息  
⚠️ **IP白名单**: 限制管理端连接来源  
⚠️ **消息签名**: 防止伪造管理员身份发送事件
