# 启动脚本问题修复报告

## 问题描述

### 现象
- 执行 `start_services.sh` 后访问 `http://192.168.66.3:8081` 返回 **404 Not Found**
- Nginx 端口与 http-server 端口冲突

### 根因分析
启动脚本同时启动了两个服务监听相同的端口：
1. **Nginx** - 监听 8080 和 8081 端口（通过 systemd）
2. **http-server** - 监听 8080 和 8081 端口（启动脚本第 150-151 行）

**导致端口冲突**：两个服务同时监听相同端口，导致 Nginx 无法正确提供服务。

## 修复方案

### 1. 修改启动脚本 `scripts/start_services.sh`

**修改前**（第 149-151 行）：
```bash
echo "[start_services] (optional) starting fallback http-servers for local testing"
start_local_http_server "$DIST_ADMIN" 8081
start_local_http_server "$DIST_USER" 8080
```

**修改后**（第 149 行）：
```bash
echo "[start_services] skipping http-servers (using nginx instead)"
```

### 2. 停止已运行的 http-server 进程
```bash
pkill -f "http-server"
```

### 3. 重新加载 Nginx
```bash
sudo systemctl reload nginx
```

## 验证结果

### 端口状态
- ✅ Nginx 监听 8080 端口（用户端）
- ✅ Nginx 监听 8081 端口（管理端）
- ✅ 无 http-server 进程运行

### 文件状态
- ✅ `/home/work/crm/dist-admin/` - 管理端文件存在
- ✅ `/home/work/crm/dist-user/` - 用户端文件存在

### 访问测试
- ✅ `http://192.168.66.3:8081` - 返回 HTTP 200 OK
- ✅ Nginx 正确提供前端文件

## 启动脚本最佳实践

### 正确的服务启动流程
1. ✅ 构建前端应用
2. ✅ 部署静态文件到 Nginx 目录
3. ✅ 安装 Nginx 配置
4. ✅ 启动后端服务（PM2）
5. ✅ 跳过 http-server（使用 Nginx 替代）

### 避免的问题
- ❌ 不要同时启动 Nginx 和 http-server
- ❌ 不要让多个服务监听相同端口
- ❌ 不要依赖 http-server 作为生产服务
