# BUG 修复报告

## 1. 问题描述

**问题现象：**
- 页面登录后仍然显示旧的"管理工具"和"其它工具"板块（这些功能已在代码中删除）
- 浏览器一直加载旧的 JS 文件，即使代码已更新并部署

**期望结果：**
- 页面应显示更新后的界面，不包含已删除的功能板块
- JS 文件应该加载最新版本

**实际结果：**
- 浏览器缓存了旧版本的 JS 文件（缓存 7 天）
- 用户看到的是旧界面

## 2. 问题原因分析

### 根本原因
Nginx 配置问题导致静态资源缓存时间过长：

1. **主要问题**：`/etc/nginx/nginx.conf` 文件中直接定义了 server 块，并且包含了静态资源缓存配置：
   ```nginx
   location ~* \.(?:css|js|svg|ico|woff2?|ttf|eot)$ {
     expires 7d;
     add_header Cache-Control "public";
     try_files $uri =404;
   }
   ```
   这导致 JS/CSS 文件被缓存 7 天（604800 秒）

2. **配置冲突**：虽然 `/etc/nginx/sites-enabled/crm.conf` 中有正确的缓存配置（不缓存 JS/CSS），但 Nginx 优先使用了 `nginx.conf` 中的配置

3. **部署脚本问题**：
   - 部署脚本没有清除云端的 `/root/crm` 旧目录
   - 没有自动部署正确的 Nginx 配置

## 3. 修复方案

### 3.1 修复 Nginx 配置

**步骤 1**：修改 `/etc/nginx/nginx.conf`，移除内嵌的 server 配置，改为引用 `sites-enabled` 目录：

```nginx
worker_processes auto;
events { worker_connections 1024; }

http {
  include       mime.types;
  default_type  application/octet-stream;
  sendfile        on;
  keepalive_timeout  65;
  client_max_body_size 2000M;

  # Include site configurations
  include /etc/nginx/sites-enabled/*.conf;
}
```

**步骤 2**：确保 `/etc/nginx/sites-enabled/crm.conf` 包含正确的缓存配置：

```nginx
# JS/CSS 文件不缓存
location ~* \.(?:js|css)$ {
  expires -1;
  add_header Cache-Control "no-cache, no-store, must-revalidate";
  try_files $uri =404;
}

# 图片字体等资源缓存 7 天
location ~* \.(?:jpg|jpeg|gif|png|svg|ico|woff2?|ttf|eot)$ {
  expires 7d;
  add_header Cache-Control "public";
  try_files $uri =404;
}
```

**步骤 3**：重启 Nginx
```bash
nginx -t && systemctl restart nginx
```

### 3.2 修复部署脚本

修改 `scripts/deploy_to_cloud.py`：

1. **清除旧目录**：在部署前删除云端 `/root/crm` 和 `/var/www/crm-*` 目录
2. **部署 Nginx 配置**：自动上传正确的 Nginx 配置文件到云端
3. **修复 nginx.conf**：确保云端 nginx.conf 正确引用 sites-enabled 配置

## 8. 技术说明

### 为什么 JS 文件名有哈希值？
Expo 构建时会生成带内容哈希的文件名（如 `index-16e17d066bacc4ffa58712866cab6743.js`），当文件内容变化时哈希值也会变化。但浏览器仍可能：
- 使用旧的 JS 文件（如果缓存未过期）
- HTML 文件仍然引用旧 JS 文件名（如果 HTML 被缓存）

### 解决方案对比
| 方案 | 优点 | 缺点 |
|------|------|------|
| 修改文件名哈希 | 浏览器自动识别新版本 | 需要 HTML 引用新文件名 |
| 不缓存 JS/CSS | 始终获取最新文件 | 增加服务器请求 |
| HTML 不缓存 | 确保引用最新 JS | 增加服务器请求 |

本方案采用**不缓存 JS/CSS + HTML 不缓存**的组合策略，确保用户始终访问最新版本。
